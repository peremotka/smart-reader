# Техническое задание (ТЗ) на разработку сервиса «Smart Reader»

## 1. Введение

Сервис «Smart Reader» представляет собой автоматизированную систему для анализа сложности англоязычных текстов и подготовки их к чтению. Цель проекта — снизить когнитивную нагрузку при изучении языка за счет предварительного ознакомления пользователя с лексикой.

## 2. Архитектура системы

Система строится на принципах микросервисной архитектуры:

- **Frontend (Client):** Интерфейс для пользователя, работающий в браузере.
- **Backend (Java/Spring Boot):** Бизнес-логика, управление пользователями, работа с БД.
- **NLP-модуль (Python):** Аналитическая подсистема для обработки и разбора текста.
- **База данных (PostgreSQL/H2):** Хранение данных пользователей и их прогресса.

## 3. Функциональные требования

### 3.1. Модуль обработки текста (Python)

- Выполнение предобработки (очистка от шума, токенизация, лемматизация).
- Определение частотности слов и их соответствия уровням CEFR.
- Формирование аналитического отчета в формате JSON.

### 3.2. Пользовательский сценарий (Frontend/Backend)

1. **Ввод данных:** Пользователь передает текст и устанавливает желаемый лимит слов для изучения (N).
2. **Этап "Pre-teaching":** Отображение списка новых слов с переводом. Возможность исключить слова, которые пользователь уже знает.
3. **Этап "Reading":** Отображение текста с визуальной подсветкой слов, не отмеченных как "изученные". Поддержка интерактивных подсказок (перевод по наведению).

## 4. Нефункциональные требования

- **Стабильность:** Обработка ошибок ввода (пустой текст, неверный формат) без падения сервиса.
- **Расширяемость:** Модульная структура, позволяющая заменять NLP-движок или формат данных без изменения бизнес-логики.
- **Производительность:** Время отклика аналитического модуля не должно превышать 3 секунд для текста объемом до 5000 символов.

## 5. Этапы реализации (Roadmap)

1. **Этап I:** Разработка базового конвейера на Python и создание API на Spring Boot (MVP).
2. **Этап II:** Интеграция базы данных, реализация пользовательского профиля и фильтрации слов.
3. **Этап III:** Полировка UX, добавление загрузки файлов и интеграция с внешними словарями.

## 6. Используемый стек технологий

- **Python:** NLTK (обработка текста), Flask/FastAPI (для связи с Java).
- **Java:** Spring Boot (серверное приложение).
- **База данных:** PostgreSQL (рекомендуется для стабильности).
- **Frontend:** HTML5, CSS3, JavaScript (React или чистый JS).
- **Система контроля версий:** Git (монорепозиторий).
